Apple 回调说明

                
Apple 登录成功后会 POST 以下字段到 redirect_uri：

                
    
                    
  • code — 用于换取 access_token 和 refresh_token
    • 
                    
  • id_token — JWT 格式的用户身份令牌
    • 
                    
  • state — 你发起授权时传入的状态值
    • 
                    
  • user — 首次授权时返回的用户信息（JSON 字符串）
    • 
                    
  • error — 授权失败时的错误码
    •

与 Flutter sign_in_with_apple 配合使用

                
Flutter 的 sign_in_with_apple 插件在不同平台上的行为不同：

                
    
                    
  • iOS / macOS — 使用原生 Apple Sign In，不需要网页回调。
    • 
                    
  • Android — 使用 Web 授权流程，Apple 会 POST 回调到 redirect_uri，必须借助后端中转回 App。
    • 
                    
  • Web — 同样使用 Web 授权流程。
    • 
                


                

                    
当前这个 GitHub Pages 页面只能作为调试展示。真正的 Android 回调请部署 workers/apple-callback.js（Cloudflare Worker 示例），它会接收 Apple POST 并 302 跳转回你的 Flutter App。

                


                
推荐流程：

                
    
                    
  1. 在 Cloudflare Workers 部署 workers/apple-callback.js
    2. 
                    
  2. 绑定自定义域名，例如 https://auth.yourdomain.com/callback
    3. 
                    
  3. Apple Developer 的 redirect_uri 填该 Worker 地址
    4. 
                    
  4. Flutter 端 webAuthenticationOptions.redirectUri 也填该地址
    5. 
                    
  5. Android AndroidManifest.xml 配置对应的自定义 scheme 接收回调
